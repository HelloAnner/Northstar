### 1.1.1 解析要求（新增约束）
- **必须解析全部 sheet 与全部字段**，不允许只解析“企业主表”或“映射字段子集”。  
- 对每个 sheet 需要记录：行数、列顺序、字段名、空列位置（含 `Unnamed` 列）。  
- 解析时保留原始顺序，为输出 1:1 对齐提供基础。  
- **不允许写死 sheet 名称**：输入可能额外包含新月份（如“2026年1月…”）或其他历史快照 sheet。  
- **兼容策略**：通过“字段结构相似度”识别 sheet 类型（批零/住餐/汇总/小微/吃穿用等），对未知 sheet 保留原样并记录日志。  
- 详细的识别/容错规则见：`prd/06_输入容错与Sheet识别规则.md`

10) 社零额（定）

### 2.2 Sheet 字段说明（目标模板）

#### A. 批零总表（17 列）
- 统一社会信用代码 / 单位详细名称 / 行业代码
- 商品销售额：本年-本月 / 上年-本月 / 本年-1—本月 / 上年-1—本月
- 零售额：本年-本月 / 上年-本月 / 本年-1—本月 / 上年-1—本月
- 衍生指标：当月增速 / 累计增速（销售额、零售额）
- 第一次上报的IP / 填报IP（留空）

#### B. 住餐总表（21 列）
- 营业额：本年-本月 / 上年-本月 / 本年-1—本月 / 上年-1—本月
- 客房收入：本年-本月 / 上年-本月 / 本年-1—本月 / 上年-1—本月
- 餐费收入：本年-本月 / 上年-本月 / 本年-1—本月 / 上年-1—本月
- 商品销售额：本年-本月 / 上年-本月 / 本年-1—本月 / 上年-1—本月
- 衍生指标：当月/累计增速（营业额）

#### C. 批发 / 零售（17 列，结构与批零总表一致）
- 在批零总表基础上按行业分流
- 末尾存在“汇总行/增速行”（空行业代码行）需要保留

#### D. 住宿 / 餐饮（21 列，结构与住餐总表一致）
- 按行业分流
- 末尾存在“汇总行/增速行”（空行业代码行）需要保留
- 模板中存在 4 个 Unnamed 列：需按模板原样保留（值在样例中与餐费累计重复）

#### E. 吃穿用（26 列）
- 基础字段 + 商品销售额/零售额的本年/上年当月与累计
- 衍生指标：当月/累计增速，吃穿用当月零售额、上年同月、增速
- 小微计算字段（当月/上年同月）
- 网络销售额字段、开业时间年/月

#### F. 小微（6 列）
- 仅保留小微企业（规模 3/4）
- 计算字段：当月零售额、上年同月零售额、增速

#### G. 吃穿用（剔除）（5 列）
- 吃穿用中被剔除企业的当月与上年同月零售额

#### H. 社零额（定）
- 存在 54 个公式单元格（模板内公式必须完全保留）
- 关键字段依赖“限上零售额汇总、限下估算、增速权重”

#### I. 汇总表（定）
- 头部为固定模板 + 4 个公式
- 行 4 为核心数据行（单位数、上报率、当月/累计零售额、四大行业增速、小微/吃穿用增速等）
- 行 10/11 生成中文摘要文本
